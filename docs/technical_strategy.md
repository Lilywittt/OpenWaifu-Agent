# 技术思路说明

## 核心问题

这个项目同时追求三件事：人物原始资产稳定、创意持续发散、下游输出稳定进入可执行生图链。

## 坍缩是怎么来的

人物资产越稳定，创意层越容易反复落回少数熟悉套路。如果没有外部变量，场景、动作、情绪组织方式和构图会逐步收缩，最后看起来像同一个模板反复变体。

## 当前解决方法

当前方案同时做两件事。第一，创意层引入实时外部采样，把外部内容当成正式输入。第二，场景设计稿吸收这条外部变量，让后续设计继续围绕它展开。这样既能发散，又能保持内容来源清楚。

## 当前链路判断

`creative` 是防坍缩的核心层，负责把“稳定角色 + 外部变量”转成设计稿。`social_post` 把人物资产和场景设计稿转成主角口吻文案。`prompt_builder` 把设计稿编译成模型可消费的 prompt，`prompt_guard` 负责最后一层回调审查，`execution` 把 prompt 注入 ComfyUI workflow 并出图。

## 当前主要风险

当前最明确的风险在 prompt 编译层：多份设计稿里的细节可能并列堆进 prompt，动作、景别、可见身体部位和服装状态一旦冲突，模型就容易产出失真的肢体或弱化关键锚点。

## 当前策略

现在的策略是保持 `creative -> prompt_builder -> prompt_guard -> execution` 的单向链路，把内容发散留在 creative，把约束裁决和稳定性控制收在 prompt 编译与回调阶段，再用真实出图结果反推 prompt 质量。
