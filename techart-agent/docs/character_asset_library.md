# 角色资产库

角色资产库保存角色长期稳定的美术标准。它的目标是让同一个角色在立绘、事件 CG、表情差分和服装差分中保持可识别。

## 基础档案

基础档案保存角色身份和气质：

- 角色 ID
- 显示名
- 年龄段
- 身份
- 性格关键词
- 说话气质
- 项目定位

这些信息帮助系统理解角色在画面里的状态。女主、好友、对手和隐藏路线角色，在镜头、表情和身体语言上的处理会不同。

## 固定视觉特征

固定视觉特征保存角色一眼能被认出的内容：

- 脸型
- 眼型和瞳色
- 发型和发色
- 体型比例
- 皮肤特征
- 重要配饰
- 角色识别点

这部分会进入提示词生成和质量检查。比如角色有固定发饰、特殊瞳色或固定发长，生成后要检查这些要素是否保留。

## 可变化内容

可变化内容保存角色在不同场景下合理变化的部分：

- 可换发型
- 可换服装
- 可换配饰
- 临时状态
- 场景适配状态

这部分支持服装差分、节日装、湿发、受伤、脸红、流泪等变化。它让角色在保持识别度的同时适配剧情场景。

## 稳定边界

稳定边界保存角色容易跑偏的位置：

- 脸部气质边界
- 体型比例边界
- 发型长度边界
- 发色边界
- 核心配饰边界
- 画风边界

边界要写具体。比如“浅色短发，发尾停在下巴到肩部之间，生成时保持短发轮廓”，比单纯写“短发少女”更能约束生成。

## 标准参考图

参考图是角色一致性的关键资产：

- 正脸参考图
- 半身参考图
- 全身参考图
- 表情参考图
- 服装参考图
- 已通过 CG 样例
- 失败样例

已通过样例用于后续参考和复跑。失败样例用于总结常见偏移，例如脸变成熟、发型过长、服装结构错误、配饰丢失。

## 服装库

服装库按套保存，与基础人设分层管理：

- 服装 ID
- 中文名
- 使用场景
- 上衣、下装、鞋袜、外套和配饰
- 颜色和材质
- 可变化部分
- 参考图
- 适合的立绘类型
- 适合的 CG 类型

夏季校服、冬季校服、便服、睡衣和祭典浴衣都应该是独立服装记录。

## 表情差分库

表情差分库保存角色常用表情：

- 默认
- 微笑
- 大笑
- 害羞
- 生气
- 委屈
- 惊讶
- 哭泣
- 冷淡
- 认真
- 慌张
- 疲惫

每个表情建议保存文字描述、参考图和使用场景。比如“害羞”可以细分为轻微脸红、移开视线、强装镇定。

## 模型资源

模型资源保存技术美术侧可调用的文件：

- 角色 LoRA 路径
- LoRA 版本
- 触发词
- 推荐权重
- 适配 checkpoint
- 推荐负面词
- 参考图路径
- 模型 hash
- 训练数据来源说明
- 质量备注

短期先用参考图和文字约束。长期用角色 LoRA 固定脸型、发型、核心配饰和整体气质。

## 人工评价

人工评价记录用户对结果的判断：

- 收藏
- 入库
- 弃用
- 弃用原因
- 人工修改说明
- 复跑效果
- 稳定 prompt
- 稳定 seed
- 工作流版本表现

收藏代表图片有继续使用价值。它可以进入角色参考图、章节素材或训练数据候选。
