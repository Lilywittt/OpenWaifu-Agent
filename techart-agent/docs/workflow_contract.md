# 工作流契约

工作流负责把美术任务转换为生成后端请求。第一版后端以本地 ComfyUI 为主，接口要保留扩展空间，后续可以接入其他图像生成服务。

## 工作流阶段

候选图阶段一次生成多张草案。目标是让用户选择构图、表情、氛围和角色状态。

参考约束候选图阶段会加入角色参考图、姿态参考或构图草图。它用于角色一致性要求更高的任务。

精修阶段接收用户选中的候选图。目标是提高脸、手、服装边缘、光影和背景细节。

局部修复阶段接收选中图和遮罩。目标是处理手、脸、发饰、衣领、道具或背景局部问题。

高清导出阶段把完成图转换为项目规格。它处理尺寸、格式、透明背景和命名。

## 运行记录

每次工作流运行都要保存：

- runId
- taskId
- workflowPresetId
- backendId
- 输入图片和参考图
- 正向提示词
- 负向提示词
- seed
- 工作流版本
- 输出图片路径
- 耗时
- 状态
- 失败原因

## ComfyUI 接入

ComfyUI 适配层应负责这些工作：

- 读取后端 endpoint
- 选择 workflow preset
- 注入提示词、参考图、遮罩和导出尺寸
- 提交任务
- 轮询历史
- 下载图片
- 写入运行记录

工作台只提交任务意图和必要输入。ComfyUI 节点、参数、文件上传和结果下载都放在工作流适配层。

## 第一版工作流

第一版建议实现四个 preset：

- `candidate_simple`
- `candidate_controlled`
- `refine_selected`
- `inpaint_region`
- `export_upscale`

候选图和精修图都要保留原始输出。导出图按项目规格另存，避免覆盖生成过程文件。
