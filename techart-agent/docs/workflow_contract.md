# 工作流契约

工作流负责把美术任务转换为本地 ComfyUI 请求。第一版以本地模型、角色资产、控制图和工作流配置为核心。

业务 Prompt 正文属于人工维护资产。工作流层只读取 prompt 文件、填充受控变量、记录 prompt 版本和运行输入。新增或修改 prompt 正文需要项目负责人确认。

## 工作流阶段

候选图阶段一次生成多张草案。目标是让用户选择构图、表情、氛围和角色状态。

参考约束候选图阶段会加入角色参考图、姿态参考或构图草图。它用于角色一致性要求更高的任务。

精修阶段接收用户选中的候选图。目标是提高脸、手、服装边缘、光影和背景细节。

局部修复阶段接收选中图和遮罩。目标是处理手、脸、发饰、衣领、道具或背景局部问题。

高清导出阶段把完成图转换为项目规格。它处理尺寸、格式、透明背景和命名。

## 运行记录

每次工作流运行都要保存：

- runId
- taskId
- workflowPresetId
- backendId
- 资产选择结果
- 输入图片、参考图、控制图和遮罩
- 模型资源、LoRA、ControlNet 和 upscale 配置
- 正向提示词文件版本和渲染结果
- 负向提示词文件版本和渲染结果
- seed
- 工作流版本
- 输出图片路径
- 耗时
- 状态
- 失败原因

## ComfyUI 接入

ComfyUI 适配层应负责这些工作：

- 读取后端 endpoint
- 选择 workflow preset
- 根据 `workflow_asset_bindings.json` 选择角色资产
- 复制本次运行需要的参考图、控制图和遮罩
- 注入提示词、模型资源、LoRA、参考图、控制图、遮罩和导出尺寸
- 提交任务
- 轮询历史
- 下载图片
- 写入运行记录

工作台只提交任务意图和必要输入。ComfyUI 节点、参数、文件上传和结果下载都放在工作流适配层。

## 资产调动

`candidate_simple` 读取面部身份、三视图、服装设定和画风参考，用于生成第一批候选图。

`candidate_controlled` 在候选资产基础上读取姿态控制和构图控制，用于更稳定的姿态、动作和镜头。

`refine_selected` 读取选中候选图、面部身份、服装设定和细节锚点，用于低幅度重绘和完成度提升。

`inpaint_region` 读取源图、遮罩、修复区域相关细节图和必要身份图，用于局部修复。

`export_upscale` 读取通过验收的图、项目导出规格和 upscale 配置，用于生成交付文件。

## 第一版工作流

第一版建议实现四个 preset：

- `candidate_simple`
- `candidate_controlled`
- `refine_selected`
- `inpaint_region`
- `export_upscale`

候选图和精修图都要保留原始输出。导出图按项目规格另存，避免覆盖生成过程文件。
