# 产品架构

`techart-agent` 面向 B 端技术美术生产。产品中心是“把项目里的角色和场景需求，稳定生产成可审阅、可修复、可入库、可导出的美术图片资产”。

## 核心对象

项目资产保存一个 galgame 项目的画风、角色、服装、表情、背景地点、导出规格和生成记录。它是所有任务的根。

角色资产库保存每个角色的长期美术标准。它记录固定视觉特征、可变化内容、参考图、服装、表情、模型资源、已通过素材和失败样例。

美术任务表示一次明确生产需求。任务绑定项目、资产类型、角色、画面说明、工作流预设和导出规格。

工作流运行记录保存一次生成后端调用。它记录输入、prompt、参考图、seed、工作流版本、输出图片、耗时和失败原因。

审图记录保存人工判断。它记录收藏、入库、弃用、问题类型、修复区域和人工备注。

导出记录保存交付文件。它记录最终图片、尺寸、格式、透明背景、命名和落点。

反馈数据保存长期改进材料。收藏图、弃用原因、人工修复记录和最终入库结果会沉淀为提示词调优、工作流调优和训练数据候选。

## 主链路

```text
项目资产
  -> 角色资产库
  -> 美术任务
  -> 工作流运行
  -> 候选图
  -> 人工审图
  -> 精修或局部修复
  -> 质量验收
  -> 入库与导出
  -> 反馈数据沉淀
```

## 模块边界

`assets` 管项目资产和角色资产库。它关心角色像谁、服装长什么样、参考图在哪里、哪个模型资源可用。

`tasks` 管任务。它关心这次要生成什么图、给哪个项目用、绑定哪些角色、采用哪个工作流、输出到什么规格。

`workflows` 管生成后端。它关心如何把任务输入转成 ComfyUI 或其他后端请求，如何拿回图片和运行记录。

`review` 管人工判断。它关心哪张图被选中、哪里需要修、为什么弃用、能否进入资产库。

`export` 管交付。它关心图片尺寸、格式、透明背景、命名规则和导出目录。

`feedback` 管改进材料。它关心哪些输入和结果值得沉淀，如何形成后续训练数据候选和工作流调优依据。

## 第一版闭环

第一版先覆盖事件 CG 和角色立绘：

1. 创建项目。
2. 建立角色资产库。
3. 创建事件 CG 或角色立绘任务。
4. 批量生成候选图。
5. 人工选择候选图。
6. 对选中图精修。
7. 对明确问题做局部修复。
8. 通过验收后入库。
9. 按项目规格导出。

这一版跑通后，再扩展背景、道具、服装差分、表情差分和批量任务组。
