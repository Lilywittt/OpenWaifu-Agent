# 审图反馈

审图反馈是技术美术工具的质量核心。用户每一次选择、收藏、弃用、框选修复和入库，都会告诉系统哪些结果值得继续加工。

## 审图动作

第一版需要支持这些动作：

- 收藏
- 入库
- 弃用
- 标记问题类型
- 框选局部问题
- 送入精修
- 送入局部修复
- 导出

## 问题类型

问题类型使用 `config/review_taxonomy.json` 管理。常见问题包括：

- 脸不像
- 发型偏移
- 体型比例偏移
- 服装错误
- 配饰缺失
- 手部错误
- 表情错误
- 构图偏离
- 背景杂乱
- 画风偏移
- 分辨率不足
- 透明背景失败

## 反馈数据

反馈数据分三类：

第一类是正样本。入库图、收藏图、最终导出图、人工确认的参考图都属于正样本。

第二类是问题样本。弃用图、失败图、局部修复前图片和问题标注都属于问题样本。

第三类是修改样本。精修前后、局部修复前后、人工备注和最终选择记录都属于修改样本。

## 后续用途

这些反馈可以用于三件事：

1. 优化提示词模板。
2. 调整工作流参数和默认 preset。
3. 整理角色 LoRA、风格 LoRA 或评审模型的训练数据候选。

每条反馈都要能回到任务、角色、资产类型、工作流版本和图片路径。这样系统后续分析质量问题时能找到来源。
