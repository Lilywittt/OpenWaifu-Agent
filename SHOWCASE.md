## 一、创作目标

OpenWaifu-Agent 面向希望持续生产角色美术资产的创作者。典型用户包括文字冒险游戏开发者、视觉小说开发者、虚拟角色创作者，以及希望为自有角色设计形象的个体玩家。

这些用户经常拥有一个角色概念，随后需要持续生成同一角色在不同剧情、不同姿态、不同服装、不同情绪和不同宣发场景下的图片资产。单次生成图片只解决一个瞬间的画面需求。角色美术生产还需要角色档案、场景设计、姿态控制、生成记录、版本复跑和结果归档共同配合，让角色资产能够长期积累和继续加工。

本作品把 AI 生图组织成一套围绕角色运行的 Agent 流程。用户可以从自然语言输入开始，也可以从实时采样开始，让系统逐步生成场景、设计稿、文案、提示词和图片结果。

## 二、核心用户与使用场景

第一类用户是游戏开发者。他们需要为角色生成立绘、剧情图、活动图、宣传图和社媒图。生成结果要能按项目和角色归档，方便后续继续修改。

第二类用户是虚拟角色创作者。他们希望角色持续出现在新的生活场景、剧情场景和社媒内容中，同时保持角色形象和口吻稳定。

第三类用户是希望设计角色形象的个人用户。他们可以先写角色书，再通过展示页和工作台逐步得到角色图、姿态图、场景图和可发布内容。

## 三、完整 Agent 链路

OpenWaifu-Agent 的主链路从用户目标或实时采样开始。

用户可以显式输入场景，例如“让角色在放学后的天台看夕阳”。用户也可以让系统从社媒、论坛和文本源里实时采样灵感。采样内容进入 Agent 后，会与角色档案结合，生成一份世界场景设计稿。世界场景设计稿描述角色所在的地点、发生的事件、情绪、心理活动和与周围世界的交互。

世界场景设计稿之后，Agent 会继续拆解成三份设计稿：

1. 环境光影设计稿：说明空间、时间、天气、灯光方向、色调和氛围。
2. 服装造型设计稿：说明角色服装、配饰、发型、材质和视觉重点。
3. 动作姿态设计稿：说明人物姿态、身体重心、手脚位置、视线方向和画面动势。

同时，系统会生成符合角色口吻的社媒文案。这样一张图既能作为游戏美术资产，也能进入角色运营和社媒发布链路。

Prompt Builder 负责把角色设定、世界场景稿和三份设计稿编译成适配 ComfyUI / Animagine XL 的正向提示词和负向提示词。Prompt Guard 会继续检查姿态逻辑、角色一致性和提示词与设计稿之间的明显冲突。检查结果进入运行记录，方便用户判断问题发生在哪个阶段。

执行层接收提示词、角色资产、姿态控制图和模型配置，调用本地 ComfyUI 工作流生成图片。每次运行都会保存输入、输出、审查结果、最终图和运行摘要。

## 四、用户如何与 Agent 协作

作品的交互方式是“自然语言输入、阶段产物审阅、用户反馈复跑”。

用户先选择角色或创建角色。角色档案中保存外貌、身材比例、年龄感、服装、性格、故事背景和参考资产。

进入任务后，用户选择从实时采样、手写场景稿、采样文本、创意包或提示词包启动。系统生成世界场景稿后，用户可以直接编辑文字。系统生成三份设计稿后，用户可以查看环境光影、服装造型和动作姿态是否符合预期。进入执行层前，用户可以选择姿态控制图，决定人物大致动作结构。

结果页会展示最终图，也展示设计稿、提示词、控制图和运行记录。用户可以收藏优秀图、复用参数、复制任务、进入下一轮修改。角色脸、服装、姿态、构图或画风出现偏差时，用户可以把具体反馈注入下一轮。

## 五、执行层与 TechArt-Agent

TechArt-Agent 是围绕 OpenWaifu-Agent 执行层拆出的本地技术美术客户端。它承接上游的角色档案、设计稿、提示词和姿态控制需求，负责本地模型调用、图片生成、资产管理和结果归档。

当前执行层重点维护四类资源：

1. 角色资产库：保存角色书、参考图、设定图和生成记录。
2. 设定图库：保存角色绑定的面部、服装、细节、三视图和其他约束图。
3. 姿态图库：保存全项目复用的 OpenPose 控制图。
4. 模型资源配置：保存 Animagine XL、ControlNet、LoRA、IPAdapter 和工作流预设。

执行层可以根据设备档位选择不同工作流。低配档用于本地验证主链路，中高配档用于更完整的角色一致性、姿态控制、局部修复和高清导出。

## 六、界面设计

展示页第一屏展示一个成功案例的完整链路，让用户直观看见最终图、工作台截图和本地客户端截图。页面中的 OpenWaifu-Agent 截图展示任务入口、起点终点选择、场景稿输入和目录审阅入口；TechArt-Agent 截图展示角色、设定图、姿态图、成品插画和设置的本地客户端结构。

工作台围绕角色组织。角色页用于创建和选择角色，设定图页用于维护角色绑定资产，姿态图页用于维护控制图库，成品插画页用于启动生成任务并查看历史，设置页用于设备档位和模型资源配置。

界面分为新手层和进阶层。新手层按步骤引导用户输入角色、编辑设计稿、选择姿态图、生成图片和回调修改。进阶层开放 LoRA、更多控制图类型、质量基准、版本归档和后训练数据沉淀。

## 八、阶段性结果

当前作品已经具备完整展示价值：用户能看到角色资产生产工具的主链路，能理解 Agent 如何从灵感采样进入场景设计稿，再进入三份设计稿、提示词、执行层和结果归档。

公网体验入口、本地测试工作台、QQ Bot 客户端和运维面板共同构成了产品形态。公网体验入口适合快速试用，本地测试工作台适合高权限调试、审阅、收藏和复跑，运维面板用于查看队列、日志和运行记录。

姿态调整能力仍在迭代。当前测试表明，OpenPose 控制能够影响人物姿态，Animagine XL 能提供较高基础画质。后续重点是进一步提升源图角色一致性，尤其是面部特征、身材比例和视觉年龄的稳定保持。

## 九、创作价值

本作品的价值在于把角色设定、灵感采样、场景设计、提示词编译、模型执行、作品归档和用户回调组织成一套持续运行的角色美术资产生产流程。它适合游戏开发、虚拟角色运营和角色视觉设计场景，也能继续扩展为 LoRA 训练、控制图管理和本地技术美术工作流的基础。
